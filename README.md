\# Dynamic Pricing Engine



A real-time dynamic pricing engine built with Kafka, Redis, FastAPI, and ML-based demand forecasting.



It simulates ride and driver events, calculates surge pricing based on demand and supply, stores the latest pricing window in Redis, and exposes live results through FastAPI endpoints.



\## Features



\- Real-time ride event generation.

\- Real-time driver event generation.

\- Kafka-based event streaming.

\- Surge pricing calculation using demand, supply, and alpha.

\- Redis-backed storage for latest pricing windows.

\- ML-based demand forecasting to update pricing parameters.

\- FastAPI endpoints for live pricing data.

\- Docker-based infrastructure for local development.

\## Project Flow



1\. Ride events are generated by the ride producer.

2\. Driver events are generated by the driver producer.

3\. Events are streamed through Kafka.

4\. The surge engine reads demand and supply data.

5\. Pricing windows are calculated every fixed interval.

6\. The latest window is stored in Redis.

7\. The FastAPI backend serves live surge data through API endpoints.

8\. The ML module updates the optimal alpha value from forecasted demand.



\## API Endpoints



\- `GET /surge/{geohash}` - returns current surge multiplier and price range.

\- `GET /latest/{geohash}` - returns the latest stored pricing window.

\- `GET /docs` - Swagger UI documentation.

\## Setup



\### 1. Start infrastructure



```cmd

cd C:\\Users\\jaski\\OneDrive\\Desktop\\dynamic\_pricing\_engine\\infra

docker compose up -d kafka zookeeper

```



If your project also uses Redis, MinIO, Grafana, or Flink through Docker Compose, start those services from the same `docker-compose.yml` as needed.



\### 2. Activate virtual environment



```cmd

cd C:\\Users\\jaski\\OneDrive\\Desktop\\dynamic\_pricing\_engine

venv\\Scripts\\activate

```



\### 3. Seed Redis



```cmd

python -m redis\_config.seed\_config

```



\### 4. Run producers



Open two separate terminals:



```cmd

python -m producers.ride\_producer

python -m producers.driver\_producer

```



\### 5. Run surge engine



```cmd

python -m flink\_jobs.surge\_engine

```



\### 6. Run demand forecasting



```cmd

python ml\\demand\_forecast.py

```



\### 7. Run FastAPI backend



```cmd

cd backend

python -m uvicorn app:app --reload

```
